Why this matters

Every chatbot answer is built one token at a time, and once you can see that loop you can tell when someone's explanation of how these systems work has skipped a step. By the end you will be able to say exactly what the model does between your prompt and its reply.

The takeaway

The model does one thing, over and over: score every possible next token against the text so far and pick one. The reply streams because it is written one pick at a time, and it varies run to run because the pick can be sampled rather than fixed.
